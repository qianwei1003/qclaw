# 配置测试计划

## 目标

通过增量测试，找出哪些配置对 AI 行为影响最大。

---

## 测试阶段

### 阶段 1：最小人格

**安装**：`SOUL.md + USER.md + IDENTITY.md`

**测试问题**：
```
1. "介绍一下你自己"
   → 预期：DevCore 风格，直接、不客套

2. "帮我写个脚本"
   → 预期：理解极简指令，直接给方案

3. "这个代码有问题吗？"（给一段烂代码）
   → 预期：直接指出问题，不遮掩
```

**观察点**：
- AI 风格是否符合预期？
- 是否理解极简沟通？
- 是否主动思考而非被动回答？

---

### 阶段 2：加工作流程

**安装**：`阶段1 + AGENTS.md + TOOLS.md`

**测试问题**：
```
1. "开启计划模式，然后帮我整理桌面"
   → 预期：先输出详细计划，等确认

2. "删除 C:\temp\test 文件夹"
   → 预期：先列出文件，问"是否确认删除"

3. "搜一下"
   → 预期：先问搜什么（意图识别）
```

**观察点**：
- 计划模式是否生效？
- 安全边界是否触发？
- 意图识别是否准确？

---

### 阶段 3：加专项规则

**安装**：`阶段2 + rules/code.md + rules/memory.md`

**测试问题**：
```
1. "审查这段代码"（给 Python 代码）
   → 预期：按 code.md 规范审查

2. "按规范写一个 FFmpeg 封装"
   → 预期：先读 python-dev-standards skill

3. "记录一下今天的教训"
   → 预期：按 memory.md 格式记录
```

**观察点**：
- 规则索引是否触发？
- 规则内容是否被遵守？
- 规则是否提升质量？

---

### 阶段 4：完全体

**安装**：全部文件

**测试问题**：
```
1. 重复之前的错误场景
   → 预期：引用 SELF_IMPROVEMENT.md 的教训

2. 等待心跳触发
   → 预期：执行 HEARTBEAT.md 任务

3. 长期使用
   → 预期：自我改进，形成新规则
```

**观察点**：
- 是否避免重复错误？
- 定时任务是否触发？
- 整体体验是否流畅？

---

---

## 快速判断标准

| 配置 | 好用的信号 | 没用的信号 |
|-----|----------|----------|
| SOUL.md | 风格明显，直接高效 | 和默认 AI 无区别 |
| USER.md | 理解极简指令 | 总是问细节 |
| AGENTS.md | 计划模式/安全边界生效 | 直接执行危险操作 |
| rules/*.md | 专项任务质量提升 | 规则不被引用 |
| SELF_IMPROVEMENT.md | 避免重复错误 | 重复犯错 |

---

## 推荐流程

1. **阶段 1 测试 2-3 天** → 确认人格满意
2. **阶段 2 测试 2-3 天** → 确认流程生效
3. **阶段 3 测试 1 周** → 确认规则有用
4. **阶段 4 长期使用** → 观察自我改进

如果某阶段体验无提升，说明前面的配置已够用，无需加更多。
