# Memory 记忆规则

---

## 通用记忆流程（所有任务适用）

### 触发条件
- 任何任务完成/失败/被纠正后
- 会话结束前

### 执行步骤
1. **即时记录**：学到新教训 → 立即记录到 memory/YYYY-MM-DD.md
2. **会话结束**：更新当日日记（今天做什么、学到什么）
3. **重要提炼**：有价值的内容 → 提炼到 MEMORY.md
4. **定期回顾**：每周/每月整理到长期记忆

### 记忆文件位置
- **每日日记**：~/.openclaw/workspace/memory/YYYY-MM-DD.md
- **长期记忆**：~/.openclaw/workspace/MEMORY.md
- **错误教训**：~/.openclaw/workspace/SELF_IMPROVEMENT.md

---

## 分层记录

| 层级 | 文件 | 内容 | 生命周期 |
|---|---|---|---|
| **流水账** | `memory/YYYY-MM-DD.md` | 今天做了什么、遇到什么问题 | 每天一个，定期压缩 |
| **教训** | `SELF_IMPROVEMENT.md` | 犯过的错、学到的教训 | 永久保留 |
| **规律** | `rules/*.md` | 固化的工作流 | 永久保留 |
| **长期** | `MEMORY.md` | 重要决策、关键信息 | 永久保留 |

---

## 记什么

### 流水账（每天）
- ✅ 今天做了哪些任务（简短）
- ✅ 遇到什么问题
- ✅ 怎么解决的
- ❌ 不记：具体代码、详细过程

### 教训（有就记）
- ✅ 犯了什么错
- ✅ 原因是什么
- ✅ 以后怎么避免

### 规律（反复出现才记）
- ✅ 某个流程出现 ≥5 次
- ✅ 提示用户固化成规则
- ❌ 不自动写进规则

### 长期（重要才记）
- ✅ 重要决策（选了 A 方案而不是 B）
- ✅ 关键信息（项目 X 用的是 Y 框架）
- ❌ 不记：日常琐事

```markdown
## [时间] 主题

### 做了什么
[一句话]

### 结果
[成功/失败/部分完成]

### 问题（如果有）
[遇到的问题]

### 教训（如果有）
[学到什么]
```

---

## 定期压缩

### 每周
- 回顾本周流水账
- 提炼有价值的 → `MEMORY.md`
- 删除琐碎的

### 每月
- 回顾 `MEMORY.md`
- 提炼规律 → 提示固化成规则

---

## 开发项目 Memory Bank（特定项目适用）

### 规则一：新项目 / 创建文档

**触发条件**
- 用户创建新项目（开发周期 ≥ 1 周，或多模块，或需跨会话持续开发）
- 用户明确说"创建文档"、"初始化 memory bank"

**执行**
使用 `memory-bank` skill 执行 `initialize memory bank`，在项目根目录创建 `memory-bank/` 及所有文件。

---

### 规则二：开发已有项目时，新会话必须先读文档

**触发条件**
新会话中涉及某个具体项目的开发任务

**执行步骤（强制，不可跳过）**
1. 检查项目根目录是否存在 `memory-bank/` 目录
2. 若存在 → 读取其中**所有** `.md` 文件，完成后再执行任务
3. 若不存在 → 提示用户"该项目尚未初始化 memory bank，是否现在创建？"
