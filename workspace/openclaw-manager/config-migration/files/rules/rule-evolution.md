# 规则自进化

---

## C. 规则修改流程

### 触发方式
用户说"改一下 xxx 规则"、"修改规则"或类似表达。

### 执行步骤
1. 我输出当前规则内容
2. 我给出修改建议
3. 等待用户确认
4. 执行修改
5. 记录到 `memory/rule-changes.md`：
   - 日期
   - 规则文件
   - 旧内容 → 新内容
   - 修改原因

---

## B. 发现规则缺失

### 机制
- 记录用户每次说话的关键词
- 某关键词出现 ≥5 次且无对应规则 → 提示用户"要新增规则吗？"

### 记录位置
`memory/keyword-stats.md`

### 格式
```markdown
## 关键词统计

| 关键词 | 出现次数 | 有无规则 | 备注 |
|---|---|---|---|
| 翻译 | 3 | 无 | |
| 审查代码 | 10 | 有 | rules/code.md |
```

---

## 规则修改记录

每次修改规则时，追加到 `memory/rule-changes.md`：

```markdown
## [日期] 修改 rules/xxx.md

### 原因
[用户反馈 / 低分记录 / 自我发现]

### 旧内容
[修改前的具体内容]

### 新内容
[修改后的具体内容]
```
