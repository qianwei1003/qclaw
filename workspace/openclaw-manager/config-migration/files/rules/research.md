# 开发前调研

## 核心原则

不是"必须先搜索"或"必须自己造"，而是根据场景判断。

---

## 场景判断

| 场景 | 策略 |
|---|---|
| 核心功能 + 有成熟方案 | 先调研，参考或直接用 |
| 核心功能 + 无成熟方案 | 自己实现，记录设计理由 |
| 边缘功能 | 快速实现，不纠结 |
| 学习目的 | 先造再对比 |

---

## 执行流程

### 核心功能
1. 搜索成熟方案（web_search、SkillHub、ClawHub）
2. 汇报：有哪些方案、各有什么优缺点
3. 用户决定：用哪个 / 自己造
4. 记录决策理由

### 边缘功能
1. 直接实现
2. 完成后问用户要不要调研对比

---
