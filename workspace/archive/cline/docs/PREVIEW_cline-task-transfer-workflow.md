# Cline 任务转发流程

## 📋 流程概述

```
你的原始需求
    ↓
我分析和优化
    ↓
生成结构化任务描述
    ↓
你审核和确认
    ↓
你复制给 Cline
    ↓
Cline 执行
    ↓
记录效果和反馈
    ↓
持续改进
```

---

## 🎯 第一步：你提交原始需求

### 格式（简单即可）

```
需求：[简单描述你要做什么]
背景：[可选，为什么要做这个]
约束：[可选，有什么限制]
```

### 示例

```
需求：创建一个用户列表组件
背景：用于显示系统中的所有用户
约束：需要支持分页、搜索、排序
```

---

## 🔍 第二步：我分析和优化

我会：
1. **理解需求** - 确认你真正需要什么
2. **识别问题** - 可能遗漏的地方
3. **提出建议** - 更好的实现方式
4. **补充细节** - 完善需求描述

### 我的分析输出

```
✅ 需求理解：
- 创建 UserList 组件
- 支持分页、搜索、排序
- 显示用户信息（ID、名称、邮箱、角色）

⚠️ 需要确认：
- 每页显示多少条？
- 搜索支持哪些字段？
- 排序支持哪些字段？
- 需要删除/编辑功能吗？

💡 建议：
- 使用 Composition API
- 提取为 useUserList composable
- 使用 TypeScript 定义类型
- 添加错误处理和加载状态
- 使用配置文件定义常量

📝 补充细节：
- 组件位置：src/components/business/UserList.vue
- 类型定义：src/types/user.ts
- API 服务：src/services/api/user.ts
- 常量定义：src/constants/user.ts
```

---

## 📝 第三步：生成结构化任务描述

我会生成一个详细的、可以直接发给 Cline 的任务描述。

### 任务模板

```markdown
# 任务：[任务名称]

## 📋 需求描述

[详细的需求描述]

## 🎯 目标

- [ ] 目标 1
- [ ] 目标 2
- [ ] 目标 3

## 📁 文件结构

需要创建/修改的文件：
- `src/components/business/UserList.vue` - 主组件
- `src/composables/useUserList.ts` - 业务逻辑
- `src/services/api/user.ts` - API 服务
- `src/types/user.ts` - 类型定义
- `src/constants/user.ts` - 常量定义

## 📋 详细要求

### 1. UserList 组件
- 使用 Vue 3 Composition API
- Props: userId (number, required)
- Emits: select (id: number)
- 功能：
  - 显示用户列表
  - 支持分页
  - 支持搜索
  - 支持排序
  - 显示加载状态
  - 显示错误状态
  - 显示空状态

### 2. useUserList Composable
- 管理用户列表状态
- 提供 fetchUsers 方法
- 提供分页、搜索、排序方法
- 处理错误和加载状态

### 3. API 服务
- getUserList(params) - 获取用户列表
- 参数：pageNum, pageSize, search, sortBy
- 返回：{ data: User[], total: number }

### 4. 类型定义
- User 接口
- UserResponse 接口
- UserListParams 接口

### 5. 常量定义
- DEFAULT_PAGE_SIZE = 10
- MAX_PAGE_SIZE = 100
- SORT_FIELDS = ['name', 'email', 'createdAt']

## ✅ 代码质量要求

- [ ] 完整的 TypeScript 类型注解
- [ ] 所有函数都有 JSDoc 注释
- [ ] 没有 any 类型
- [ ] 完善的错误处理
- [ ] 遵循项目结构规范
- [ ] 遵循命名规范
- [ ] 没有硬编码值
- [ ] 没有重复代码

## 🧪 测试要求

- [ ] 单元测试覆盖率 > 80%
- [ ] 测试用户列表获取
- [ ] 测试分页功能
- [ ] 测试搜索功能
- [ ] 测试排序功能
- [ ] 测试错误处理

## 📚 文档要求

- [ ] 组件有 JSDoc 注释
- [ ] 方法有 JSDoc 注释
- [ ] 复杂逻辑有行注释
- [ ] 有使用示例

## 🎨 样式要求

- [ ] 使用 scoped 样式
- [ ] 使用 CSS 变量
- [ ] 响应式设计
- [ ] 遵循 BEM 命名规范

## 📊 验收标准

- [ ] 代码通过 ESLint 检查
- [ ] 代码通过 TypeScript 检查
- [ ] 所有测试通过
- [ ] 代码质量检查通过
- [ ] 符合企业级代码标准
```

---

## ✅ 第四步：你审核和确认

你需要：
1. **阅读任务描述** - 确认是否符合你的需求
2. **提出修改意见** - 如果有不对的地方
3. **确认无误** - 说"可以执行"或"确认"

### 你的反馈示例

```
✅ 看起来不错，就这样执行

或者

⚠️ 需要调整：
- 分页大小改为 20
- 需要添加删除功能
- 需要添加批量操作
```

---

## 🚀 第五步：你复制给 Cline

你把我生成的任务描述复制到 Cline 窗口，Cline 就会按照要求执行。
