# Cline 工作流规则

> 增量修改工作流 - 避免双份代码问题

## 核心原则

1. **只给改动部分** - 用行号标记位置，不给整个文件
2. **逐个确认** - 多文件时逐个展示，逐个确认
3. **改动前预览** - >3 行或多文件时等用户确认
4. **透明化** - 方案 + 代码片段同时给

## 代码展示格式

**正确格式**（推荐）：

```
文件：src/app.js
改动位置：第 15 行

// 改为：
const x = 2;  // 原来是 1
```

**错误格式**（避免）：
```diff
- const x = 1;
+ const x = 2;
```
> Cline 会显示成双份完整代码

## 执行规则

| 改动量 | 操作 |
|--------|------|
| ≤3 行 | 直接执行 |
| >3 行 | 等用户说"执行/确认/yes" |
| 多文件 | 逐个文件确认 |

## 触发时机

- 用户说"给我代码"
- 用户说"先不执行"
- 用户说"让我看看"
- 用户说"review 一下"
- 任何代码改动前

## 检查清单

- [ ] 是否只给了改动片段（不是整个文件）
- [ ] 是否用"第 X 行"标记了位置
- [ ] 是否等待用户确认（>3行时）
- [ ] 用户说"不/取消"时是否停止
