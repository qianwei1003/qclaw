## 已记录改进

### 2026-03-23 忘记计划模式

### 问题/不足
用户说"执行计划模式"，我直接执行了操作，没有先输出计划

### 原因分析
执行时没有先检查计划模式状态

### 改进方案
1. 用户说"执行计划模式"后，所有操作都要先输出计划
2. 在 AGENTS.md 中强化这个规则
3. 每次执行前检查是否开启了计划模式

### 应用场景
- 所有需要执行的任务

### 2026-03-22 学习 ClawHub Skills 安装

### 问题/不足
ClawHub API 有严格的 Rate Limit，一次安装太多 Skills 会导致全部失败

### 原因分析
不理解 API 限流机制，批量安装导致 120/120 配额快速耗尽

### 改进方案
1. 一次只安装 1-2 个 Skill
2. 每次安装间隔 2-3 分钟
3. 使用 `clawhub inspect` 查看 Skill 信息（不消耗安装配额）
4. 对于可疑 Skill，先 inspect 确认安全再安装

### 应用场景
- ClawHub Skills 安装
- 任何有 API 限流的操作

### 2026-03-22 理解用户"计划模式"需求

### 问题/不足
用户希望在工作前先看到详细的执行计划，不接受黑箱操作

### 原因分析
之前没有充分理解用户想要透明度的需求

### 改进方案
创建了 PLAN_TEMPLATE.md 和执行计划模式，让用户了解每一步操作

### 应用场景
- 所有需要执行的任务
- 用户要求"计划模式"时

### 2026-03-22 多智能体并行工作

### 问题/不足
用户问如何让多个 AI 智能体同时工作

### 原因分析
需要更灵活的多会话/多任务处理能力

### 改进方案
1. 使用 sessions_spawn 创建独立会话
2. 使用 subagents 进行后台并行任务
3. 不同会话可以设置不同人格

### 应用场景
- 需要同时处理多个任务时
- 复杂任务分解并行处理
