# 测试策略

## 测试优先级

| 优先级 | 内容 | 说明 |
|--------|------|------|
| P0 | 核心操作 | trim、concat、remove_silence、remove_static |
| P1 | 边界情况 | 空视频、极短/极长视频、无音轨视频 |
| P2 | 性能 | 处理速度、内存占用 |

## 测试原则

1. 每个操作在真实视频上跑一遍，记录输入输出
2. 边界条件有明确预期（不是"没报错就算成功"）
3. FFmpeg 命令错误（如 -c copy 失败）要有 fallback 测试
4. 每添加一个操作，同步添加对应的测试用例

## 当前测试视频

| 文件 | 时长 | 特点 |
|------|------|------|
| `tests/1.mp4` | 195s | 幻灯片风格，大量静止帧 |

## 操作测试矩阵

| 操作 | 测试输入 | 预期输出 | 验证方式 |
|------|---------|---------|---------|
| `trim_start` | start_time=10 | 删掉前10秒 | 时长对比 |
| `trim_end` | end_time=60 | 删掉60秒之后 | 时长对比 |
| `trim_range` | start=0, end=60 | 仅保留0-60秒 | 时长==60s |
| `trim_range` 多段 | segments=[(0,10),(30,50)] | 仅保留20秒 | 时长==20s |
| `concat` | 2个视频 | 合并后时长=两个之和 | 时长对比 |
| `remove_silence` | 1.mp4 | 有静音段被删除 | 时长减少 |
| `remove_static` | 1.mp4 | 静止段被删除 | 时长减少（1.mp4从195s→93s） |
| `convert` | 1920x1080→1280x720 | 输出分辨率==1280x720 | ffprobe 验证 |
| `info` | 任意视频 | 返回元数据 | 字段完整 |

## 验证方式

每次操作后用 ffprobe 验证：
- 时长是否在预期范围内（±5%容差）
- 分辨率是否符合预期
- 文件大小是否大于 1KB（排除空文件）
