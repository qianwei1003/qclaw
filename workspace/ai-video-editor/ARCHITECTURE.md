# AI 视频剪辑工具 - 架构规划

**文档版本：** 1.0  
**更新日期：** 2026-03-24

---

## 📐 架构分层

```
┌─────────────────────────────────────────────────────────────┐
│                    通用基础设施层                              │
│  (所有阶段都会用到的功能)                                     │
├─────────────────────────────────────────────────────────────┤
│  • Parser（需求解析）- 自然语言 → 结构化指令                  │
│  • Executor（执行器）- 调用 FFmpeg 执行视频处理               │
│  • Validator（验证器）- 验证输出视频是否正确                  │
│  • 视频下载 - yt-dlp / 工具集成                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    工具层                                    │
│  (封装外部工具，统一的 Python 接口)                           │
├─────────────────────────────────────────────────────────────┤
│  • FFmpeg - 视频处理基础                                    │
│  • auto-editor - 自动剪辑静音/静止                          │
│  • PySceneDetect - 场景检测                                 │
│  • Whisper - 语音识别/字幕                                  │
│  • OpenCV - 内容识别                                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    业务阶段层                                │
├─────────────────────────────────────────────────────────────┤
│  V1: 基础自动剪辑                                           │
│  V2: 场景分割                                               │
│  V3: 自动字幕                                               │
│  V4: 智能剪辑                                               │
└─────────────────────────────────────────────────────────────┘
```

## 📋 V1: 基础自动剪辑

**目标：** 自动删除静音/静止片段

### 核心功能
| 小功能 | 工具 | 状态 |
|--------|------|------|
| 静音检测 | auto-editor | ⏳ |
| 静止检测 | auto-editor | ⏳ |
| 删除静音段 | FFmpeg | ⏳ |
| 删除静止段 | FFmpeg | ⏳ |
| **多片段截取合并** | FFmpeg | ⏳ **新增** |

### 验收标准
- [ ] 能够处理一个测试视频
- [ ] 输出删除静音后的视频
- [ ] 处理时间在可接受范围内

### 技术方案
```
用户: "删除静音段"
   ↓
Parser → {"operation": "remove_silence", "params": {...}}
   ↓
Executor + auto-editor → 找出静音段
   ↓
Executor + FFmpeg → 删除静音段
   ↓
Validator → 验证输出
   ↓
完成
```

## 💡 关键原则

1. **通用基础设施优先** - Parser/Executor/Validator 完成后可被所有阶段复用
2. **工具封装独立** - 每个外部工具单独封装，不耦合业务逻辑
3. **阶段解耦** - 每个阶段独立开发，可单独测试
4. **增量开发** - 每个阶段的小功能可独立完成和测试
