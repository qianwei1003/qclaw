# Task Dispatcher Skill

多智能体任务分发器。根据任务类型自动派发给合适的智能体，或支持手动指定。

## 功能

1. **关键词自动匹配** - 根据任务内容自动选择合适的智能体
2. **手动 @ 指定** - 使用 @ 指定特定智能体
3. **多智能体协作** - 复杂任务派发给多个智能体
4. **子智能体创建** - 一个智能体可以创建多个子代理并行处理

## 关键词映射

| 关键词 | 智能体 |
|--------|--------|
| 爬、抓取、采集、搜索 | crawler |
| 代码、编程、写程序、开发、调试、bug | coder |
| 分析、统计、数据、对比、趋势 | analyst |
| 写、文档、整理、报告、润色、改写 | writer |
| 研究、调研、调查、了解 | researcher |

## 使用方式

### 1. 关键词自动匹配

```
"帮我爬取 TDesign 组件文档"
→ 自动匹配 crawler 智能体
```

### 2. 手动 @ 指定

```
"@coder 帮我写个排序算法"
→ 指定 coder 智能体执行
```

### 3. 多智能体协作

```
"帮我研究 Vue 3 并整理成报告"
→ 自动判断需要多智能体协作
→ 流水线：爬虫 → 分析师 → 写作
```

### 4. 并行执行

```
"对比 React, Vue, Angular 的优缺点"
→ 派发多个爬虫并行执行
→ 汇总结果
```

## 协作模式

### pipeline（流水线）

```
爬虫 → 分析师 → 写作 → 最终报告
```

### parallel（并行）

```
爬虫A → 结果A ─┐
爬虫B → 结果B ─┼─→ 汇总
爬虫C → 结果C ─┘
```

### discussion（讨论）

```
coder（代码角度） ─┐
analyst（性能角度）┼─→ 汇总建议
writer（可维护性）─┘
```

## 内部实现

- 使用 `sessions_spawn` 派发任务给子代理
- 使用 `Promise.all` 并行执行多个子代理
- 通过共享文件传递数据（写入 workspace）
