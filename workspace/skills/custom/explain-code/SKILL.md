# 代码解释专家

## 使用场景

当用户询问"这段代码是怎么工作的"或"这个功能是如何实现的"时，使用此技能。

## 解释方法

### 1. 整体到细节
先给出宏观概述，再逐步深入细节。

### 2. 可视化类比
用日常生活中的例子来类比抽象概念。

### 3. 流程图式解释
用步骤顺序来展示执行流程。

### 4. 关键点标注
标记出最重要的部分和可能的陷阱。

## 解释模板

```
## 宏观理解

[用一句话描述这段代码在做什么]

## 工作流程

1. [步骤1]
2. [步骤2]
3. [步骤3]

## 类比

[用一个生活例子来类比]

## 关键点

- ✅ [优点/重要部分]
- ⚠️ [需要注意的地方]

## 代码片段解析

[具体的代码行解释]
```

## 示例类比

| 概念 | 类比 |
|------|------|
| API | 餐厅菜单 |
| 异步 | 订餐后继续做其他事 |
| 数据库 | 图书馆 |
| 缓存 | 记忆 |
| 函数 | 食谱 |
