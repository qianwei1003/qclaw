# 代码审查专家

## 使用场景

- 用户要求审查代码
- 提交代码前自查
- 发现 bug 根源
- 优化代码质量

## 审查维度

### 1. 正确性
- [ ] 逻辑错误
- [ ] 边界条件
- [ ] 空指针/未定义
- [ ] 类型错误

### 2. 安全性
- [ ] SQL 注入
- [ ] XSS 漏洞
- [ ] 敏感信息泄露
- [ ] 权限问题

### 3. 性能
- [ ] 时间复杂度
- [ ] 空间复杂度
- [ ] 不必要的循环
- [ ] 重复计算

### 4. 可维护性
- [ ] 代码重复
- [ ] 函数过长
- [ ] 命名不清
- [ ] 缺少注释

### 5. 最佳实践
- [ ] 错误处理
- [ ] 日志记录
- [ ] 测试覆盖
- [ ] 文档完整

## 审查模板

```
## 代码审查报告

### 基本信息
- 语言：
- 行数：
- 复杂度：

### 发现的问题

#### 🔴 严重 (必须修复)
1. [问题描述]
   - 位置：[文件:行号]
   - 建议：[修复方案]

#### 🟡 中等 (建议修复)
1. [问题描述]
   - 位置：[文件:行号]
   - 建议：[修复方案]

#### 🟢 轻微 (可选优化)
1. [问题描述]
   - 位置：[文件:行号]
   - 建议：[优化建议]

### 总体评价
- 可读性：⭐⭐⭐⭐⭐
- 安全性：⭐⭐⭐⭐⭐
- 性能：⭐⭐⭐⭐⭐
- 推荐分数：X/Y

### 修复优先级
1. [最紧急的修复]
2. [次要优化]
3. [可选改进]
```

## 常用检查项

### JavaScript/TypeScript
- 严格模式
- async/await 错误处理
- 依赖注入
- 类型检查

### Python
- PEP 8 规范
- 类型提示
- 异常处理
- 虚拟环境

### HTML/CSS
- 语义化标签
- 响应式设计
- 无障碍访问
- CSS 性能
