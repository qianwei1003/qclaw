# 决策规则库

## 核心原则

遇到问题时，按照以下顺序尝试解决方案：

---

## 1. 文件操作

| 情况 | 处理方式 |
|------|----------|
| 文件不存在 | 检查路径是否正确，询问用户是否需要创建 |
| 文件已存在 | 询问用户是覆盖还是追加 |
| 删除文件 | 绝对禁止！先列出文件，确认后再操作 |
| 批量操作 | 先列出受影响文件，确认后再执行 |

---

## 2. 命令执行

| 情况 | 处理方式 |
|------|----------|
| 命令不存在 | 检查是否安装，尝试其他方法 |
| 权限不足 | 提示用户权限问题，建议解决方案 |
| 执行超时 | 增加 timeout 或分步执行 |
| Rate Limit | 等待一段时间后重试（通常 30-120 秒） |
| 网络错误 | 检查网络，重试 |

---

## 3. API 调用

| 情况 | 处理方式 |
|------|----------|
| 401 未授权 | 检查 API Key 是否正确 |
| 403 禁止 | 检查权限设置 |
| 429 Rate Limit | 等待后重试，指数退避 |
| 500 服务器错误 | 重试 2-3 次，仍失败则报告 |
| 网络错误 | 检查网络，重试 |

---

## 4. 任务执行

| 情况 | 处理方式 |
|------|----------|
| 复杂任务 | 分解为小步骤，每步确认 |
| 多步骤任务 | 先输出计划，用户确认后再执行 |
| 后台任务 | 告知用户任务在后台运行 |
| 长时间任务 | 定期报告进度 |

---

## 5. ClawHub Skills 安装

### 安装决策树

```
开始安装
    ↓
检查 Rate Limit 状态
    ↓
如果 Rate Limit → 等待 60-120 秒后重试
    ↓
检查 Skill 是否已安装
    ↓
如果已安装 → 跳过，告知用户
    ↓
检查 Skill 安全标记
    ↓
如果有可疑标记 → 询问用户是否强制安装
    ↓
执行安装
    ↓
验证安装结果
```

### 安装规则

1. **一次只安装 1 个 Skill**
2. **安装间隔至少 30 秒**
3. **优先使用 inspect 查看信息**
4. **失败后等待 60+ 秒再重试**
5. **记录成功/失败状态**

---

## 6. 学习与改进

| 情况 | 处理方式 |
|------|----------|
| 失败后成功 | 记录有效的方法到 DECISION_RULES.md |
| 用户纠正 | 立即记录到 SELF_IMPROVEMENT.md |
| 发现更好方法 | 更新相关配置文件 |
| 重复错误 | 立即停止并反思 |

---

## 决策检查清单

执行任何操作前，先问自己：

1. **这个操作可逆吗？** → 不可逆的操作要格外小心
2. **用户知道会发生什么吗？** → 模糊指令要确认
3. **有更好的方法吗？** → 思考后再执行
4. **失败了我该怎么办？** → 准备好备选方案
5. **这个操作会影响什么？** → 考虑副作用

## 常用命令速查

| 场景 | 推荐命令 |
|------|----------|
| 安装 Skill | `clawhub install <name>` |
| 查看 Skill 信息 | `clawhub inspect <name>` |
| 搜索 Skill | `clawhub search <query>` |
| 检查目录 | `dir` (Windows) / `ls` (Linux) |
| 读取文件 | `read` |
| 写入文件 | `write` |
| 编辑文件 | `edit` |
| 执行命令 | `exec` |
