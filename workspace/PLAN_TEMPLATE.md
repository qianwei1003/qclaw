# 执行计划模板

## 使用说明

当用户提出需要执行的任务时，使用此模板先输出执行计划，等待用户确认后再执行。

---

## 计划输出格式

```
==========================================
执行计划
==========================================

【步骤 N】具体命令/操作
├─ 工具：工具名称 (read/write/exec/browser等)
├─ 操作：具体做什么
├─ 参数/目标：操作的目标
└─ 预期结果：执行后的结果

...

【总览】
工具：工具名称 × N
修改文件：[文件列表]
创建文件：[文件列表]

确认执行？回复"执行"或"确认"
==========================================
```

## 示例

### 示例1：提交代码到 GitHub

```
==========================================
执行计划
==========================================

【步骤 1】git add .
├─ 工具：exec (shell command)
├─ 操作：添加所有文件到暂存区
├─ 目标：当前工作目录
└─ 预期：所有非忽略文件被暂存

【步骤 2】git commit -m "提交信息"
├─ 工具：exec (shell command)
├─ 操作：创建提交
├─ 参数：-m "提交信息"
└─ 预期：生成新的 commit 对象

【步骤 3】git push origin master
├─ 工具：exec (shell command)
├─ 操作：推送到远程仓库
├─ 参数：origin master
└─ 预期：代码推送到 GitHub

【总览】
工具：exec × 3
修改文件：无
创建文件：无

确认执行？回复"执行"或"确认"
==========================================
```

```

```

---

## 触发条件

当用户提出以下类型的任务时，自动触发此模板：
- 执行命令
- 创建/修改/删除文件
- 推送/拉取代码
- 发送消息
- 调用 API
- 任何需要"动手"的操作

**例外**：纯对话、问答、解释概念等不需要执行的操作不需要输出计划。

---

## 确认机制

- 用户回复"执行"、"确认"、"yes"、"y" → 开始执行
- 用户回复"取消"、"不"、"no"、"n" → 取消执行
- 用户修改计划 → 重新输出计划
- 用户提问 → 先回答问题，再决定是否执行
