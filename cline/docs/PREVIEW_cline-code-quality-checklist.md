# Cline 代码质量检查清单

## 📋 使用说明

这个清单用于在 Cline 生成代码后进行质量检查。可以在以下场景使用：

1. **生成新组件后** - 检查组件是否符合规范
2. **生成工具函数后** - 检查函数是否完整
3. **代码审查时** - 系统地检查代码质量
4. **重构前** - 识别需要改进的地方

---

## ✅ 类型安全检查

### TypeScript 类型检查
- [ ] 所有函数都有返回类型注解
- [ ] 所有参数都有类型注解
- [ ] 没有使用 `any` 类型
- [ ] 没有使用 `unknown` 而不进行类型守卫
- [ ] 所有对象都有接口定义
- [ ] 所有数组都指定了元素类型
- [ ] 可选属性使用 `?` 标记
- [ ] 只读属性使用 `readonly` 标记

### Vue 组件类型检查
- [ ] Props 都有类型定义
- [ ] Props 都有默认值（如果不是 required）
- [ ] Props 都有验证器（如果需要）
- [ ] Emits 都有类型定义
- [ ] Emits 都有验证器
- [ ] 模板中的数据都有类型
- [ ] 计算属性都有返回类型

---

## 📝 注释和文档检查

### JSDoc 注释
- [ ] 所有导出的函数都有 JSDoc
- [ ] JSDoc 包含 @param 标记
- [ ] JSDoc 包含 @returns 标记
- [ ] JSDoc 包含 @throws 标记（如果会抛出异常）
- [ ] JSDoc 包含 @example 标记（复杂函数）
- [ ] JSDoc 描述清晰准确

### 行注释
- [ ] 复杂逻辑有行注释
- [ ] 行注释解释"为什么"而不是"是什么"
- [ ] 没有过度注释（如 `const x = 1 // 定义变量 x`）
- [ ] 没有过时的注释

### 组件文档
- [ ] 组件有 name 属性
- [ ] 组件有 JSDoc 注释
- [ ] 组件的 props 有描述
- [ ] 组件的 emits 有描述
- [ ] 组件有使用示例

## 📊 检查结果

### 优秀（90-100%）
- 所有检查项都通过
- 代码质量高
- 可以直接合并

### 良好（80-89%）
- 大部分检查项通过
- 有少量改进空间
- 需要小幅修改

### 一般（70-79%）
- 一半以上检查项通过
- 有明显改进空间
- 需要重新审视

### 需要改进（<70%）
- 大量检查项未通过
- 需要重大改进
- 建议重新生成

## 🎯 使用建议

1. **第一次使用**: 逐项检查，理解每个检查项的意义
2. **日常使用**: 快速扫描，重点关注关键项
3. **代码审查**: 系统检查，确保代码质量
4. **持续改进**: 定期回顾，优化检查流程
